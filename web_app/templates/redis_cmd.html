常用命令示例

命令说明
KEYS *列出所有 Key（谨慎使用，大数据量会阻塞）
SCAN 0游标方式遍历 Key（推荐替代 KEYS）
DBSIZE当前数据库 Key 数量
INFO serverRedis 服务器信息
GET key名获取指定 Key 的值
TYPE key名查看 Key 的类型
TTL key名查看 Key 的剩余生存时间
LLEN queue:xxx查看队列长度
LRANGE queue:xxx 0 10查看队列前 10 条
HGETALL event:meta:{fp}查看事件元数据
SMEMBERS event:set事件去重集合元素数
FLUSHDB清空当前数据库（仅 fs3:* 数据，危险）
FLUSHALL清空所有数据库（极度危险）

Redis 持久化命令

命令说明
BGSAVE后台异步保存 RDB 快照到磁盘（推荐）
SAVE同步保存 RDB 快照（阻塞，不推荐）
LASTSAVE查看最后一次成功保存的时间戳
INFO persistence查看持久化状态（rdb_last_save_time 等）
CONFIG GET dir查看 RDB 文件存储目录
CONFIG GET dbfilename查看 RDB 文件名（默认 dump.rdb）
CONFIG SET save ""禁用自动 RDB 快照（临时）
CONFIG SET save "900 1"恢复自动快照策略

恢复流程：
1. 确认 Redis 正在运行 → PING
2. 查看 RDB 目录和文件名 → CONFIG GET dir + CONFIG GET dbfilename
3. 停止所有 FlowScan 写入 → Worker 暂停
4. 保存当前快照 → BGSAVE
5. 关闭 Redis → SHUTDOWN NOSAVE
6. 替换 RDB 文件 → cp /path/to/backup/dump.rdb <dir>/<dbfilename>
7. 重启 Redis → redis-server 或 systemctl restart redis
8. 验证 → DBSIZE 检查键数量

清空后恢复流程：
1. BGSAVE（先备份当前状态）
2. FLUSHDB（清空当前库）
3. 如需恢复：替换 RDB 文件后重启 Redis

注意：RDB 恢复是全量替换，会覆盖当前所有数据。建议先 BGSAVE 备份。
